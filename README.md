#  Data Storytelling: Uma Análise, Três Audiências

Este projeto foca em uma das habilidades mais raras e valorizadas em dados: a **tradução de insights**. 

O objetivo não é apenas gerar gráficos, mas construir narrativas específicas para diferentes níveis de decisão dentro de uma organização, utilizando exatamente a mesma base de dados.

---

##  O Conceito
Dados são apenas ruído se não forem interpretados corretamente para o público certo. Neste repositório, demonstro como transformar a análise de **Otimização de Estoque** em três versões distintas:

### 1. Visão Executiva (C-Level)
- **Foco:** Decisões estratégicas e rentabilidade.
- **Narrativa:** "Quanto dinheiro estamos ganhando ou perdendo?"
- **Visualização:** Dashboards de faturamento, impacto de devoluções e ROI.

### 2. Visão Operacional (Logística/Gestores)
- **Foco:** Processos, fluxo e eficiência diária.
- **Narrativa:** "Onde os itens estão e como otimizar o espaço?"
- **Visualização:** Boxplots de custos de armazenamento e giro de SKU por categoria.

### 3. Visão Técnica (Engenharia de Dados/Data Science)
- **Foco:** Metodologia, estatística e integridade.
- **Narrativa:** "Como os dados foram tratados e quais as limitações do modelo?"
- **Visualização:** Histogramas de distribuição, análise de outliers e correlações.

##  Tecnologias Utilizadas
- **Python**: Processamento e análise.
- **Matplotlib & Seaborn**: Customização visual de acordo com a audiência.
- **Pandas**: Consolidação das fontes de dados.

##  Estrutura do Repositório
- `Storytelling_Across_Audiences.ipynb`: O coração do projeto, demonstrando a mudança de narrativa.
- `inventory_data.csv`, `products_catalog.csv`, `sales_data.csv`: Bases de dados utilizadas.

---
**Diferencial do Portfólio:** Este projeto mostra maturidade profissional ao provar que o autor entende que a análise de dados serve ao negócio e deve ser comunicada com clareza para todos os stakeholders.
